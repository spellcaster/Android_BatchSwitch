**Скрипт для пакетного управления (включения/выключения) компонентов установленных Android приложений**

Позволяет пакетно управлять компонентами (intents, receivers, services) установленных Android приложений. Достаточно один раз создать список компонентов (либо получить готовый), после чего процесс отключения компонентов будет занимать несколько секунд.

ОС: Windows

Требования
----------

- На ПК: установленный и запущенный сервис ADB
- На устройстве: root права процессу ADB

Применение
----------

- Открыть консоль команд
- Убедиться, что ADB доступен (текущая директория = пути к `adb.exe` либо путь к `adb.exe` добавлен в `PATH`)
- Вызвать `process.bat` `e|d` `путь-к-файлу-списка`

Параметры
---------

- `e|d` - включить (enable) / отключить (disable) компоненты
- `путь-к-файлу-списка` - путь к файлу со списком компонент

Файл списка
-----------

Должен содержать по одному компоненту вида `код пакета`/`код компонента` (например, `com.google.android.gms/gcm.GcmDiagnostics`) на строку. Пустые строки или строки, начинающиеся с `#`, считаются комментариями и игнорируются.

Примеры списков
---------------

- `GoogleServices.lst` - списки для Google Play Services и Google services framework. Закомментированы (**не** отключаются) компоненты обновления системы и компоненты Push (WhatsApp, VK и прочие мессенджеры); раскомментируйте по желанию.

- `YandexApps.lst` - списки для программ Yandex: Метро, Расписание, Навигатор, Маркет, Такси. Отключены метрики, социальная аутентификация.
